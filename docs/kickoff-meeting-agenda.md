# 慢病随访系统 — 第一次全员启动会议程

> 日期：2026年7月__日 | 时长：约90分钟 | 形式：线下/飞书会议

---

## 会议目标

统一所有人对项目的理解，避免后续理解偏差、意见冲突、开发返工。

---

## 议程（90分钟）

| 时间 | 环节 | 内容 | 主讲 |
|---|---|---|---|
| 0:00-0:10 | 开场 | 项目背景、挑战杯赛制、评分标准简介 | 组长 |
| 0:10-0:25 | 系统全貌 | 五大模块演示（患者→随访→预警→统计→管理）、全链路数据流讲解 | 组长 |
| 0:25-0:40 | 技术规范 | 统一技术栈（Vue3+Spring Boot+MySQL）、25个接口清单、数据库6张表、统一返回格式 | 组长 |
| 0:40-0:55 | Git协作 | 分支策略（main/develop/个人分支）、每日工作流（pull→分支→commit→push→PR→merge）、提交信息规范 | 组长 |
| 0:55-1:10 | AI辅助开发 | 规范Prompt写法、CodeArts工具介绍、代码开发完整流程、规避AI幻觉的方法 | 组长 |
| 1:10-1:20 | 分工确认 | 6人角色分配确认（B/C/D/E/F）、各角色第1周具体任务下发 | 全员 |
| 1:20-1:30 | Q&A | 组员提问、安装软件排障、确认下次会议时间 | 全员 |

---

## 会前准备（每个人）

- [ ] 通读 PROJECT_STARTUP_GUIDE.md（项目启动指南）
- [ ] 看自己角色的标准文档（前端看 FRONTEND_STANDARDS.md，后端看 BACKEND_STANDARDS.md）
- [ ] 确认电脑能装的软件清单（Node.js / JDK / MySQL / Git / VS Code / Postman）

---

## 会后行动

1. 组长：将任务拆分表导入飞书看板
2. 全员：2天内装好所有软件，在群里发截图打卡
3. 组长：建 GitHub 仓库，邀请组员，推送项目骨架
4. 组员：clone 仓库到本地，每人切一个测试分支提交一次，确认 Git 流程跑通
